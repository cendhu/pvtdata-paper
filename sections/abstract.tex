\begin{abstract}
Blockchain technology provides a decentralized and immutable ledger, but its inherent transparency poses a significant barrier to enterprise adoption in domains requiring data privacy. Existing solutions like channels provide data isolation but are coarse-grained, requiring a separate ledger and consensus instance per channel. In this paper, we introduce \textit{private data collections}, a mechanism for managing private data on permissioned blockchains. Our approach decouples private data storage from the replicated ledger: private data is shared only among authorized peers, while cryptographic hashes are recorded on the ledger for tamper-evidence and serializability verification. Central to the design is \textit{blind validation}: all nodes can verify the correctness of private transactions using only the on-chain hashes, without ever seeing the private data itself. We design a two-phase gossip-based protocol for proactive dissemination and reactive retrieval of private data, and we formalize the collection abstraction with seven design requirements. We implement our solution within Hyperledger Fabric, leveraging its simulate-order-validate-commit architecture, and demonstrate that private data collections add minimal overhead to transaction throughput while providing fine-grained data privacy. Hyperledger Fabric has shipped the mechanism in production since v1.2, with continued enhancements through v2.5.
\end{abstract}
